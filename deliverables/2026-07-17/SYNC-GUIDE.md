# 跨机同步操作指南（CSMS 项目 · 2026-07-17 v3 单文件聊天框版）

> **v2 → v3 变更**：v2 假设"拷整个文件夹"，但聊天框发不了文件夹。
> **v3 改为单个 `.md` 文件**：你只需把 `SYNC-<日期>.md` 一个文件发到 B 电脑的项目聊天框，B 的 WorkBuddy 一读就自动还原，**无需拷文件夹、无需跑脚本**。

## 核心原理
WorkBuddy 不"还原快照"，而是"读上下文续跑"——启动读 `MEMORY.md` + 日志，**不主动读 `deliverables/`**。
所以"无缝"的关键是 **记忆文件**（当日日志 / 项目 MEMORY / 用户 MEMORY）必须回到 B 电脑的原位置。

单文件同步包 `SYNC-<日期>.md` 用唯一分隔符把**所有需要同步的内容**（记忆 + 当日文档）序列化进**一个文件**：
- `§§F|<项目相对路径>|§§ ... §§/F§§` → 写回项目根下的该相对路径
- `§§U|§§ ... §§/U§§` → 写回 B 本机 `~/.workbuddy/MEMORY.md`
- 文件第一行是给 WorkBuddy 的 **执行协议注释**，任何收到此文件的 WorkBuddy 都会按指示展开。

## A → B（你现在的场景，最简单）
1. 在 A 电脑，AI 已生成（或你让我生成）`deliverables/2026-07-17/SYNC-2026-07-17.md`。
2. **把这个 `.md` 文件发到 B 电脑 CSMS 项目的聊天框**（当作附件/文件发，不要粘贴正文，否则分隔符可能被聊天框改写）。
3. 在 B 电脑对 WorkBuddy 说一句："请按这个同步包还原" —— B 的 WorkBuddy 读文件后自动把记忆写回原位置、文档重建到 `deliverables/2026-07-17/`。
4. 验证：问 B 的 WorkBuddy"E5 rev2 是什么？"能答「5 角色 + accessibleCompanies 行级权限」即成功。

## B → A（B 也工作了一天后回传）
因为规则已固化在用户级 `~/.workbuddy/MEMORY.md`（跨电脑生效），**B 电脑的 WorkBuddy 也会在每次收尾自动生成单文件包**。
1. B 收尾时，让 B 的 WorkBuddy 生成 `SYNC-<同日>.md`（或你手动让我生成）。
2. 把 B 的 `SYNC-<同日>.md` 发回 A 电脑聊天框。
3. 在 A 电脑说"请按这个同步包还原" → A 的 WorkBuddy 展开 B 的更新。
> 双向完全对称，都只发**一个 `.md` 文件**，不用文件夹、不用 U 盘、不用脚本。

## 生成命令（AI 自动或手动）
```
node scripts/sync-package-single.cjs 2026-07-17
```
- 输出：`deliverables/<日期>/SYNC-<日期>.md`
- 安全：仅收集记忆文件与当日文档；**`SECRETS.md`（凭证）永不进包**（每台电脑各自维护）。

## 习惯固化（AI 自动执行）
- 每次实质性工作收尾，AI 自动生成当天的 `SYNC-<日期>.md` 单文件包，方便你随时发到另一台电脑。

## ⚠️ 双向同步的固有风险
若 A、B **同一天都改了同一文件**（如都改了 MEMORY.md），后还原的会覆盖先的。本方案面向"一台主用、另一台滞后"场景，通常单向为主，风险低；如需保险，还原前先备份目标机的同名文件。

## 一句话记忆
> 单文件同步包：生成 `SYNC-<日期>.md` → **发到另一台电脑聊天框** → 说"请按同步包还原"。A↔B 双向只发一个文件。
