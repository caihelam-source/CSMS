## 一、中央信息库 Company Hub（条目 9）— 优先级 P0

### 1.1 功能描述
统一入口，聚合某家公司相关的全部信息：工商信息、董事/秘书/股东、关联人员、全部文档（按分类）、会议、任务、合规提醒。提供：
- **关系图谱视图**：复用 `EquityGraph` 的穿透能力，扩展到"人—公司—文件—任务"多维关系。
- **自然语言查询入口**（轻量版）：基于结构化字段的关键词检索（如"查匯駿控股 2025 年 NAR1"），不依赖外部 LLM；高级语义检索列入高级-1。

### 1.2 用户故事
- 作为公司秘书，我希望在一个页面看到某公司全部档案，而不用在 5 个菜单间跳转。
- 作为经理，我希望点开一家公司就能看到它的股权结构和关键人物关系图。

### 1.3 验收标准
- 进入 Company Hub 能看到该公司 6 类文档、董事/股东/秘书、近期会议、未完成任务、待办合规提醒。
- 关系图谱能展开到至少 3 层股权穿透。
- 关键词搜索能在 500ms 内返回匹配结果。

### 1.4 测试用例
- **TC1（正向）**：打开"匯駿控股"Company Hub → 显示其 4 类文档共 6 份、董事 3 人、未完成任务 2 条；点击"股权架构"展开 3 层穿透正常。
- **TC2（边界）**：打开一家无任何关联数据的新公司 → Hub 各分区显示"暂无数据"空态，不报错、不白屏。
- **TC3（异常）**：搜索框输入超长字符串（2000 字）或 SQL 注入式字符 → 系统做转义后返回空结果，不抛 500。

## 二、智能模板引擎 DocGen（条目 10）— 优先级 P1

### 2.1 功能描述
在现有 `Templates.jsx`（模板 CRUD + 变量填充渲染预览）基础上增强：
- **变量系统**：内置变量 `{公司名称}{公司英文名称}{注册号}{董事姓名}{日期}{财政年度}` + 自定义变量；支持变量来源自动绑定（选公司后自动填充）。
- **一键生成**：渲染完成后调用 `documentService.create` 将生成文档归档到对应公司（带 `category: 'generated'`、`type: 'template_output'`），并回显 `docNumber`。
- **模板库分类**：按用途分类（决议/通知/会议纪要/政府表格）。

### 2.2 用户故事
- 作为秘书，我希望选一个"年度大会通知"模板、选公司、填几个变量，点一下就生成正式文件并归到公司文档里。

### 2.3 验收标准
- 模板编辑支持 `{{变量}}` 占位符；预览时变量被真实值替换。
- "生成文档"按钮调用归档接口，成功后公司文档列表出现该文件且带编号。
- 强制约束：未选公司或变量未填全时，"生成"按钮禁用（与痛点 3 一致，不算随意完成）。

### 2.4 测试用例
- **TC1（正向）**：选"董事会决议"模板 → 选公司"匯駿控股" → 变量自动填充 → 点生成 → 公司文档新增 1 份 `GEN-BOARD-0001`，详情可打开。
- **TC2（边界）**：模板含 3 个变量但只填 1 个 → "生成"按钮禁用，提示"请填写全部变量"。
- **TC3（异常）**：归档接口返回 500 → 前端提示失败且不产生半成品文件，可重试。

## 三、微信小程序签署系统（条目 11）— 优先级 P2（规格 + 可行性，非本库实现）

### 3.1 功能描述（规格）
- **多模式身份验证**：微信手机号授权 + 人脸核身（微信 OCR/活体）+ 短信验证码，三选一或组合。
- **签署流程**：签署任务从 Web 端 `SignTask` 推送到小程序 → 用户查看文档 → 身份认证 → 手写/电子签名 → 提交。
- **法律合规存证**：签署完成生成含时间戳、哈希、操作日志的存证包，可对接司法区块链（如天平链/信锚链）或本地哈希存证。

### 3.2 可行性说明（诚实）
- 需要**独立小程序工程**（WXML/WXSS/JS + 微信开发者工具）、**微信开放平台**企业账号、后端存证服务。
- 本 WorkBuddy 会话**无小程序构建/真机预览能力**，无法 one-shot 交付可运行小程序。
- 本报告给出**数据模型与接口契约**，作为独立子项目立项；Web 端 `SignTask` 模型已存在，可复用其 `signers`/`status` 字段。

### 3.3 数据模型（建议）
```
SignTask（已有）: _id, document, signers[{user, status, signedAt, authMode}], status
SignEvidence（新增）: _id, signTaskId, signerId, authMode, idHash, docHash, timestamp, blockchainTxId?
```

### 3.4 测试用例（验收口径，供子项目参考）
- **TC1（正向）**：签署人手机号授权 + 人脸核身通过 → 签名提交 → `SignEvidence` 写入，`SignTask.status` 变 `partially_signed`→`fully_signed`。
- **TC2（边界）**：人脸核身连续失败 3 次 → 锁定并提示转人工核验，不产生存证。
- **TC3（异常）**：断网提交 → 本地缓存草稿，恢复后自动重试，不丢签署动作。

## 四、高级功能集

### 4.1 智能文档管理（高级-1）
- **版本控制（✅ 可）**：Document 模型加 `versions[]`（历史内容/文件 + 操作人 + 时间），前端 Documents 详情页可查看/回滚。
- **OCR（⚠️ 外部）**：接入腾讯云/百度云 OCR，上传图片自动提取文本存 `extractedText`；需 API Key，单列配置项。
- **自然语言搜索（⚠️ 外部）**：接入向量检索（如 MongoDB Atlas Vector Search 或独立向量库）；轻量版先用结构化关键词检索（见 Company Hub）。

### 4.2 合规提醒自动化（高级-2，✅ 可）
- 关键日期追踪：基于 `ComplianceRule.dueDaysBefore` + 公司财年自动生成提醒（已有 `generateReminders` 底座）。
- 自定义规则：用户可配"提前 N 天 + 每周一推送"。
- 推送：站内铃铛 + 邮件（需 SMTP 配置）；先落地站内，邮件留接口。

### 4.3 会议与决议管理（高级-3，✅ 可增量）
- 模板库：预置"年度大会/董事会/特别决议"纪要模板，接入 DocGen 变量。
- 任务分配：会议纪要生成后一键拆解为 `Task` 分配给相关人员（复用 `taskService.create`）。

### 4.4 合规风险预警（高级-4，✅ 可）
- 风险评分：逾期提醒数 × 权重 + 即将到期密度 → 公司风险等级（高/中/低），Dashboard 红/黄/绿标识。

### 4.5 数据可视化（高级-5，✅ 可）
- Dashboard 增补：任务完成率趋势、文档分类分布饼图、合规提醒时间轴。

### 4.6 多语言支持（高级-6，✅ 可增量）
- 引入 i18n（react-i18next），先抽 Dashboard/Companies 文案做中/英，逐步覆盖。

### 4.7 移动端适配（高级-7，✅ 可增量）
- 关键页面（Dashboard/Companies/Documents/Tasks）补全移动断点布局，导航改底部 Tab。

### 4.8 高级功能测试用例示例
- **TC-A（合规自动化正向）**：新建规则"年报提前 60 天、每周一推送" → 系统周一生成 3 条提醒，站内铃铛 +1。
- **TC-B（版本控制边界）**：文档回滚到第 2 版 → 当前内容变为第 2 版，且新增一条"回滚"版本记录，不丢失历史。
- **TC-C（风险预警异常）**：某公司同时有 5 条逾期提醒 → 风险等级=高，面板红色高亮，不卡死渲染。
