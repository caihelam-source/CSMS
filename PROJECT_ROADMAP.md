## 📝 如何参与

### 新团队成员 Onboarding 流程

#### 第 1 天：环境准备
1. 阅读 [README.md](./README.md) 了解项目概况
2. 按照 [DEVELOPMENT.md](./DEVELOPMENT.md) 配置开发环境
3. 克隆代码仓库，安装依赖
4. 成功启动项目，能访问 http://localhost:5173

#### 第 2-3 天：代码熟悉
1. 阅读 [CONTRIBUTING.md](./CONTRIBUTING.md) 了解贡献规范
2. 通读 `server/models/` 了解数据模型
3. 通读 `client/src/pages/` 了解页面结构
4. 尝试在本地运行各个功能模块
5. 提出至少 3 个关于代码的问题

#### 第 4-5 天：小任务上手
1. 从 Issue 列表挑选一个 "good first issue"
2. 创建分支，开始编码
3. 提交 PR，接受 Code Review
4. 根据 Feedback 修改并合并

#### 第 1 周：独立开发
1. 参与每日站会（如有）
2. 领取正式的开发任务
3. 按照敏捷流程推进工作

### 任务认领方式

1. 在项目管理工具（GitHub Projects/GitLab Boards）查看待办任务
2. 评论想要认领的任务，等待分配
3. 或直接创建分支开始开发（先提 Issue 说明）

### 代码提交规范

参考 [CONTRIBUTING.md](./CONTRIBUTING.md#git-提交信息规范) 部分。

## 💬 讨论与决策记录

### 2026-06-30: 技术选型讨论
**决定**: 继续使用 MongoDB 而非迁移至 PostgreSQL
**理由**:
- 当前数据模型已深度使用嵌套文档（如 appointments）
- 迁移成本高，收益不明确
- 团队对 MongoDB 更熟悉

### 2026-06-28: 前端框架评估
**决定**: 保持 React + Vite，暂不考虑 Next.js
**理由**:
- 当前项目是 SPA，不需要 SSR
- Vite 的 HMR 体验已经足够好
- 迁移成本 > 收益
